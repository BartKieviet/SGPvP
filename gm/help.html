The Scorpion Guard's Better PvP Script V32 User's Manual

Quick start · Configuration

Quick start

This script adds a number of keyboard commands to help in PvP combat. In general, it's used as follows:

  1. Configure the script.

    Press the Configure key (I in the default keymap) to bring up the configuration dialogue, and specify there which pilots are to be targeted, as well as your ship's armour and any custom key bindings you'd like.

  2. Set a retreat tile.

    Move to your retreat tile and press the Set retreat tile key (Z in the default keymap) to have the script "remember" it. The retreat tile can be a planet, wormhole, whatever, but it has to be in sight and reachable from the tile where PvP will occur.

  3. Engage.

    Move to the tile where PvP will occur and press the Engage key (X in the default keymap) repeatedly. As soon as a target appears, you will be taken to the combat screen. Pressing the Engage key again will fire against the target.

  4. Disengage.

    When you have to retreat, press the Disengage key (C in the default keymap) repeatedly if needed, until you are back at your retreat tile.

There are many additional functions that you can use; see below.

Configuration

Targeting

The script requires that you specify which ships on your tile are to be attacked, and which are to be excluded from attack.

The way this is done is by pasting a quick list into the configuration dialogue. The script parses the same QL format used for ambushes, and learns from it which factions, alliances, and individuals are to be included and excluded. Note, however, that friend/foe, bounty, and ship class/size criteria in the QL are ignored.

On top of the QL, the script provides for include and exclude override lists. You can specify only pilot names or IDs in these lists, not alliances or factions. Pilots excluded this way won't be engaged no matter what. Pilots included in this way will not only be engaged, but will also be prioritised: of all the ships on your tile, the one listed first on your include override list will be targeted.

When no priority targets are present on the tile, then a target is selected based on ship type — the script attempts to engage traders and battleships first, breakers last. As a last resort, all else being the same, the pilot with the lowest ID will be engaged.

Keymap

In the configuration dialogue, click on Key bindings to view and edit the keymap. You may click on any key to customise its behaviour. Do note that there many available functions not bound to a key by default, so explore and suit yourself.

If you remove the Configure key, or change its binding and later forget what you chose, remember that you can always press Esc to open the configuration dialogue again.

Robots

You have to specify your ship's maximum armour points and installed armour strength (1 through 6). This is required for computing the amount of robots to use when repairing your ship. Note that the script will repair you to full armour even if it has to partially waste one robot. If you'd rather the script doesn't waste bots, manually configure your ship armour points to the result of:

max_ship_armour - (180/armour_strength) + 1

(e.g. 445 for a Hawk with level V armour)

There are rare situations in which the script cannot ascertain how many robots are available, or the state of the ship's armour; the script will then guess, based on last known values, but it may get it wrong. For such cases, you may want to define and use Force use robots keys: the script will attempt to use as many bots as you specify for those, regardless of what it "thinks" you have or are needed.
